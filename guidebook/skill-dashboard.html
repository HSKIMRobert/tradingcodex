Primary skill

tcx-dashboard

Get a compact read-only overview of the current attached workspace: attention items, recent research, forecasts, portfolio and order posture, pending permissions, broker state, and the best viewer destination for detail.

Ask for the overview you need #

Use the skill when you want to know what changed, what needs attention, where work stands, or what to inspect next. With no narrower request, it checks the smallest useful set of current workspace domains.

Example

$tcx-dashboard
Show me what changed, what needs attention, and where to inspect it.

Read recorded state, not guesses #

The overview puts explicit blocked, waiting, stale, failed, unhealthy, pending, uncertain, or incompatible states first. It calls something changed only when a trusted comparison, version, event, or timestamp establishes that change; missing data remains unknown.

Available detail is routed to #/library, #/skills, or #/system in the read-only viewer.

Keep overview separate from action #

Dashboard is not analysis or execution. It does not begin an analysis run, dispatch roles, create artifacts, make investment recommendations, mutate workspace or service state, or perform an order action.

Use tcx-workflow for a new investment judgment. Use tcx-server when service readiness or recovery is the problem.
