Optional integration

Use OpenBB without making it a hidden dependency

TradingCodex reuses governed data first, then tries relevant user-added capabilities, the supported OpenBB integration, and finally reviewed official sources. Source order never replaces source-quality checks.

Understand the source order #

  1. Reuse. Consume an exact current-run Dataset or Snapshot only after its authenticated acquisition receipt confirms coverage and source.
  2. Respect user capabilities. Try one relevant enabled user MCP or skill.
  3. Use OpenBB. Query one explicit provider through the optional supported integration.
  4. Fall back officially. Use regulator, central-bank, or public statistical sources and disclose exchange-price gaps where no reviewed free official source exists.
Priority is not trust. An unofficial or partial result stays screen-grade and may still need primary-source corroboration. Every fallback preserves its predecessor receipt, and only the exact missing coverage moves forward.

Use keyless official data by default #

When the higher tiers cannot satisfy a DataNeed, evidence-producing roles can query the built-in SEC, U.S. Treasury, BLS v1, ECB, World Bank, CFTC, and Bank of Canada adapters. TradingCodex constructs only reviewed official-host requests, returns at most 120 normalized rows, and immediately stores the result as a Snapshot, Dataset, and acquisition receipt. It never exposes a raw response body to the handoff.

Partial means stop and preserve. The first usable partial or reference result is recorded before a role requests only the exact missing fields, identifiers, or period. Free-key official sources remain an explicit setup gap rather than silently reading an environment variable.

Let tcx-server guide setup #

Ask $tcx-server to inspect OpenBB status and explain the configured provider slot—or identify a naming-convention hint that still needs provider-documentation confirmation—together with the access declaration and restart step. The skill never asks you to paste an API key. It hands the actual change to your workspace terminal.

Ask in Codex

$tcx-server
Help me connect my free FMP API key to the supported OpenBB integration.
Run in the workspace terminalshell
./tcx data-sources openbb provision
./tcx data-sources openbb configure fmp \
  --access free \
  --credential-ref fmp_api_key=env:FMP_API_KEY
./tcx data-sources openbb enable fmp \
  --data-kind equity_price \
  --auto-use allow
./tcx update --skip-refresh --no-doctor

Put the secret value only in the environment that launches Codex. The command stores the environment-variable reference, never its value. After the update finishes, fully quit and restart Codex, then open a new task; restarting only the Django service does not refresh the OpenBB MCP projection or credentials.

Check state without spending quota #

Local statusshell
./tcx data-sources openbb status --json

Status separates the user-declared access tier, credential-reference availability, isolated runtime compatibility, workspace projection, restart requirement, and last observed provider result. A key does not prove that an endpoint is free or entitled. A keyless provider stays in ask mode until a bounded probe has observed it as callable.

Use probe only when you intentionally want a minimal provider call, because it may consume quota:

./tcx data-sources openbb probe fmp --data-kind equity_price --symbol AAPL

Restart the right process #

Credential and provider changes are frozen when an OpenBB MCP process starts. Open a new task, and fully restart Codex if the existing process is reused. ./tcx service stop and ensure restart the Django viewer service; they do not reload OpenBB credentials.

OpenBB remains optional. If the latest upstream runtime cannot be verified or fails compatibility checks, TradingCodex quarantines it and continues to the official-source fallback instead of silently using an older version.

Keep provider terms visible #

OpenBB is a transport, not a data license or accuracy guarantee. TradingCodex records and cross-checks the requested, returned, and actual upstream provider, source timing, adjustment policy, warnings, compatibility receipt, and retention/redistribution posture with the resulting Snapshot and Dataset. Unofficial and personal-use sources require explicit secondary-source consent.

Read the canonical data-source and OpenBB reference for the complete source matrix, API surfaces, and licensing boundary.
