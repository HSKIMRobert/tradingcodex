Use OpenBB without making it a dependency

TradingCodex keeps source routing in one short skill, preserves provenance, and never stores a provider credential.

Use the source order #

  1. Reuse. Start with a relevant SourceSnapshot or Dataset already supplied for the work.
  2. Use one relevant user capability. An enabled Skill, Plugin, or MCP can answer the missing question.
  3. Use OpenBB if enabled. It is a direct optional MCP for evidence roles.
  4. Prefer original records. Use company IR and filings, exchanges, central banks and statistics agencies, or regulators.
  5. Then use reliable web sources. If the fact is still unavailable, state the data gap.
Keep partial results. Do not repeat an unchanged source request. Retain the useful part and ask the next source only for the missing identifier, field, or period.

Preserve the right evidence #

Every used external source becomes a SourceSnapshot with its provider, locator, timing posture, hash, warnings, and needed excerpt. Create a Dataset only for reusable tables, time series, OHLCV, or financial tables—and keep every row you use. A research artifact cites the exact Snapshot and Dataset IDs.

Documents do not need a Dataset. A filing, news item, IR release, or qualitative web page normally needs only its SourceSnapshot.

Use the default optional projection #

OpenBB is enabled by default as a non-required source, but it is not installed or started during attach or update. The generated configuration lets Codex invoke the official OpenBB MCP directly with uvx when an evidence role actually uses it. TradingCodex stores only environment-variable names; put their values in the environment that starts Codex.

Run in the workspace terminalshell
./tcx data-sources openbb status
./tcx data-sources openbb env add FMP_API_KEY
./tcx update --skip-refresh --no-doctor

Use disable to turn the projection off; that explicit choice survives later updates. Use enable to restore it. Restart Codex after a projection or environment change.

Keep the boundary clear #

OpenBB is projected only to fundamental, technical, news, macro, instrument, and valuation roles. Head Manager, portfolio, risk, and judgment roles do not receive raw OpenBB tools. TradingCodex does not install, configure, proxy, approve, or audit a user capability or OpenBB.

OpenBB and its providers remain third-party services. Licensing, provider terms, data rights, cost, entitlement, and accuracy are your responsibility; direct optional use does not resolve AGPL or downstream data-license duties. Read the canonical data-source and OpenBB reference for details.
