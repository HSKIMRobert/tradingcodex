Help & status

Check the workspace without changing it

Use the dashboard skill for a current workspace overview and the server skill for readiness, update, MCP, and recovery diagnostics.

Read the skill pages #

01tcx-dashboardSee what needs attention and where to inspect it.→02tcx-serverCheck readiness and receive safe recovery guidance.→
Primary

tcx-dashboard

Summarize current workspace attention items, recent work, portfolio/order posture, forecasts, permissions, broker state, and viewer destinations.

Use it when
You want to know what changed, what needs attention, where work stands, or what to inspect next.
What it does
Uses only the smallest relevant canonical read-only status and ledger queries, with explicit as-of posture.
What it returns
A compact overview and links to the relevant Library, Skills, or System viewer section.
Boundary
It does not begin analysis, dispatch roles, create artifacts, make recommendations, or change state.

Review the workspace

$tcx-dashboard
Show me what changed, what needs attention, and where to inspect it.
Primary

tcx-server

Explain workspace and service health, update readiness, MCP state, database posture, and recovery guidance.

Use it when
The workspace is not ready, you need the viewer URL, MCP needs checking, or you want a safe recovery and update posture before a new task.
What it does
Uses the smallest trusted read-only status surface and interprets hook startup context plus read-only status/update tools.
What it returns
Runtime status, a recovery command, a viewer URL, update guidance, or a clearly stated blocker.
Boundary
It does not perform investment judgment, run service commands through the model shell, modify connector code, or make a workspace change.

Check readiness

$tcx-server
Is this workspace ready for a new analysis?

Use the terminal for explicit operations #

When a service, doctor, update, or recovery command requires the launcher, the skill gives you the exact user-terminal handoff rather than running it through the model shell. That keeps maintenance operations clearly operator-owned.

Common readiness checkshell
./tcx doctor
./tcx service ensure

On native Windows PowerShell, use .\tcx.cmd doctor and .\tcx.cmd service ensure.

What the viewer can show #

The local workspace viewer is a read-only projection. It can show Library, Skills, and System information for registered attached workspaces, but it never launches or resumes Codex, creates an analysis run, modifies a skill, or offers a browser exception to policy and execution boundaries.

Use a new native Codex task for agent work. The viewer is where you inspect the workspace; the trusted Codex workspace is where research and managed skills run.
